How is the result generated?

The site requests a uniformly random 32-bit value from crypto.getRandomValues(), the browser's cryptographically secure random number generator. The final bit selects heads or tails, giving each result an equal probability.

The site does not use Math.random(), contact a server, or send flip results anywhere. If secure randomness is unavailable, flipping is disabled.

No software-only website can prove literal physical randomness. This implementation provides strong, unbiased digital randomness suitable for normal decisions and games.

Runs locally in your browser. No tracking. No external dependencies.
